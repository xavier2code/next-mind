    Create a new `docker-entrypoint.sh` at project root. Per decisions D-01, D-03 (DOCK-03).

    Use the exact content from RESEARCH.md Code Examples:

    ```bash
    #!/bin/sh
    set -e

    # Wait for PostgreSQL to be ready (D-03)
    DB_HOST="${DB_HOST:-postgres}"
    DB_PORT="${DB_PORT:-5432}"
    DB_USER="${POSTGRES_USER:-nextmind}"
    MAX_RETRIES=30
    RETRY_INTERVAL=2

    echo "Waiting for PostgreSQL at ${DB_HOST}:${DB_PORT}..."
    attempt=0
    until pg_isready -h "$DB_HOST" -p "$DB_PORT" -U "$DB_USER" -q; do
      attempt=$((attempt + 1))
      if [ $attempt -ge $MAX_RETRIES ]; then
        echo "ERROR: PostgreSQL did not become ready after ${MAX_RETRIES} attempts"
        exit 1
      fi
      echo "  Attempt ${attempt}/${MAX_RETRIES} - PostgreSQL not ready, waiting..."
      sleep $RETRY_INTERVAL
    done

    echo "PostgreSQL is ready. Running Drizzle migrations..."
    npx drizzle-kit migrate

    echo "Starting Next.js application..."
    exec "$@"
    ```

    Key points:
    - Shebang is `#!/bin/sh` (Alpine has busybox sh, NOT bash)
    - `set -e` exits on any error
    - Uses `pg_isready` (available in postgres:16-alpine image, but also available via the npm-installed drizzle-kit's postgres dependency in the runner stage). If pg_isready is NOT available in the Node 22-alpine runner, the fallback is to install postgresql-client: `RUN apk add --no-cache postgresql-client` in the Dockerfile runner stage.
    - Retries 30 times with 2-second intervals (60s total timeout)
    - Runs `npx drizzle-kit migrate` which reads drizzle.config.ts and applies all migrations from ./drizzle/
    - `exec "$@"` replaces the shell process with CMD from Dockerfile (`node server.js`)
    - Uses env vars with defaults: DB_HOST=postgres (Docker service name), DB_PORT=5432, POSTGRES_USER=nextmind

    After creating the file, make it executable:
    ```bash
    chmod +x docker-entrypoint.sh
    ```

    **IMPORTANT:** If pg_isready is not available in the alpine runner image, the Dockerfile (Plan 02) will need `RUN apk add --no-cache postgresql-client` in the runner stage. The executor should verify this and add it to the Dockerfile if needed. However, since the Dockerfile was created by Plan 02, only modify it if absolutely necessary (add a single RUN line, do not restructure).
