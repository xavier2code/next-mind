    **Step 2: Generate fresh migration**
    This requires DATABASE_URL to point to a real PostgreSQL database. The local .env file or .env.local should have DATABASE_URL. Run:
    ```bash
    npm run db:generate
    ```
    If DATABASE_URL is not set or the database is unreachable, the command will fail. The migration generation reads schema.ts and generates SQL -- it does NOT apply changes to the database.

    If `npm run db:generate` fails because drizzle-kit tries to diff against a non-existent database, an alternative approach is acceptable:
    - Temporarily set DATABASE_URL to any valid PostgreSQL URL
    - Or use `drizzle-kit generate --force` if available in v0.31.10

    **Step 3: Verify the generated migration**
    The generated SQL file (whatever name drizzle-kit assigns, e.g., 0000_*.sql) MUST contain:
    - 13 CREATE TABLE statements for: user, session, account, verificationToken, conversation, message, audit_log, agent, workflow, task, agent_message, file, conversation_file
    - The workflow table MUST have a `checkpoint` jsonb column
    - All foreign key constraints from schema.ts
    - All indexes from schema.ts

    Verify with:
    ```bash

    ```

    **IMPORTANT:** Do NOT run `npm run db:migrate` or `npm run db:push`. Only generate the migration files. Applying migrations is the Docker entrypoint's job (D-03).
