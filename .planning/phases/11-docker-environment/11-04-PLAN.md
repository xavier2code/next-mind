  <action>
    Build the Docker images and fix any errors that arise. Per DOCK-01, DOCK-07.

    **Step 1: Verify Docker is available**
    ```bash
    docker --version && docker compose version
    ```
    If Docker is not installed, this is a BLOCKER. Output a clear message asking the user to install Docker Desktop, Colima, or OrbStack, and STOP.

    **Step 2: Build the app image**
    ```bash
    docker compose build app 2>&1
    ```

    **Step 3: If build fails, diagnose and fix:**

    Common failure scenarios and fixes:

    a) **Turbopack standalone output error**: If `npm run build` fails with Turbopack standalone issues, modify the Dockerfile builder stage build command:
    - Change: `npm run build`
    - To: `npx next build --webpack`
    - Per D-05 fallback strategy

    b) **npm ci fails**: Check package-lock.json exists and is valid. If missing, run `npm install` locally first.

    c) **Missing native dependencies on Alpine**: If npm install fails with node-gyp errors for any package:
    - Add `RUN apk add --no-cache build-base python3` in the deps stage
    - Per RESEARCH.md Pitfall 5, this is LOW risk for this project's pure-JS dependencies

    d) **COPY fails**: Check .dockerignore is not excluding needed files. The Dockerfile copies: package.json, package-lock.json, source code, drizzle/, drizzle.config.ts, public/, docker-entrypoint.sh. None of these should be in .dockerignore.

    **Step 4: Verify image was created**
    ```bash
    docker images | grep nextmind
    ```

    **Step 5: Check image size**
    ```bash
    docker images nextmind-docker-environment-app --format "{{.Size}}"
    ```
    Should be in the ~200MB range. If significantly larger (>500MB), investigate:
    - Full node_modules copied to runner (should use standalone's pruned copy)
    - Unnecessary packages installed in runner stage
    - Large files in public/ directory
  </action>
  <verify>
    <automated>docker images | grep -E "nextmind|docker" | head -5</automated>
  </verify>
  <acceptance_criteria>
    - `docker compose build app` exits with code 0
    - `docker images` shows the built image
    - Image size is under 500MB (target ~200MB)
  </acceptance_criteria>
  <done>Docker image builds successfully; any build errors fixed; image size reasonable</done>
</task>

  <action>
    Start the Docker environment and verify all success criteria. If any step fails, fix the issue and retry.

    **IMPORTANT: Before starting**, ensure `.env.docker` has a real AUTH_SECRET value (min 32 characters). If AUTH_SECRET is empty, generate one:
    ```bash
    AUTH_SECRET=$(openssl rand -base64 32) && sed -i '' "s/^AUTH_SECRET=$/AUTH_SECRET=$AUTH_SECRET/" .env.docker
    ```

    **Step 1: Start the environment**
    ```bash
    docker compose up -d
    ```
    Wait for both containers to be running (check with `docker compose ps`).

    **Step 2: Check container logs for migration success**
    ```bash
    docker compose logs app
    ```
    Look for these log lines in order:
    1. "Waiting for PostgreSQL at postgres:5432..."
    2. "PostgreSQL is ready. Running Drizzle migrations..."
    3. "Starting Next.js application..."

    **Step 3: Verify login page loads**
    ```bash
    curl -f http://localhost:3000/login
    ```
    Expected: HTTP 200 with HTML response containing login form.

    **Step 4: Verify database tables**
    ```bash
    docker exec nextmind-postgres psql -U nextmind -d nextmind -c "\dt"
    ```
    Expected: 13 tables listed (user, session, account, verificationToken, conversation, message, audit_log, agent, workflow, task, agent_message, file, conversation_file).

    **Step 5: Verify volume persistence (uploads)**
    ```bash
    docker exec nextmind-app ls -la /app/data/uploads
    docker compose restart app
    docker exec nextmind-app ls -la /app/data/uploads
    ```
    Expected: /app/data/uploads directory exists after restart.

    **Step 6: Check image size**
    ```bash
    docker images | grep nextmind
    ```
    Expected: App image ~200MB range.

    If any step fails, diagnose and fix:
    - Migration failure: Check entrypoint logs, verify DATABASE_URL in .env.docker matches docker-compose.yml
    - App crash: Check `docker compose logs app` for Node.js errors
    - Login page 500: Check app logs for database connection errors
    - Missing tables: Verify drizzle/ migration files were copied correctly, check migration SQL
  </action>
  <verify>

  </verify>
