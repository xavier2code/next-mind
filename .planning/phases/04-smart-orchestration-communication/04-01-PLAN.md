<feature>
  <name>Wave-based Task Scheduler</name>
  <files>src/lib/agents/scheduler.ts, tests/lib/agents/scheduler.test.ts</files>
  <behavior>
    Given a list of tasks with optional dependencies:
    - Tasks with no dependencies are grouped into Wave 0 (can run in parallel)
    - Tasks that depend on Wave 0 tasks are grouped into Wave 1
    - Pattern continues until all tasks are assigned to waves
    - Circular dependencies throw CircularDependencyError
    - Waves execute sequentially, tasks within wave execute in parallel with MAX_CONCURRENCY=3

    Cases:
    - [A, B, C] with no deps -> single wave with all 3 tasks
    - [A->B, C] -> Wave 0: [A, C], Wave 1: [B]
    - [A->B->C] -> Wave 0: [A], Wave 1: [B], Wave 2: [C]
    - [A->B, A->C, B->D, C->D] -> Wave 0: [A], Wave 1: [B, C], Wave 2: [D]
    - [A->B, B->A] -> throws CircularDependencyError
    - [A, B, C, D, E] with no deps -> executes in chunks of 3
  </behavior>
  <implementation>
    1. Create TaskWithDependencies interface extending Subtask with id and dependencies fields
    2. Create ExecutionWave interface with waveIndex and taskIds
    3. Implement computeExecutionWaves function using Kahn's algorithm for topological sort
    4. Implement WaveScheduler class with executeWaves method
    5. Use Promise.all for parallel execution within waves, chunked by MAX_CONCURRENCY
    6. Implement cascade cancel: when task fails, mark dependent tasks as cancelled
  </implementation>
</feature>

  <action>
    Create src/lib/agents/scheduler.ts with the following exports:

    1. TaskWithDependencies interface:
       - Extends Subtask with id: string and dependencies: string[] = []
       - Export type for use in decomposition

    2. ExecutionWave interface:
       - waveIndex: number
       - taskIds: string[]

    3. ScheduledTask interface:
       - task: TaskWithDependencies
       - waveIndex: number
       - status: 'pending' | 'running' | 'completed' | 'failed' | 'cancelled'
       - result?: SubAgentExecutionResult

    4. CircularDependencyError class:
       - Extends Error with message "Circular dependency detected in task graph"

    5. computeExecutionWaves function:
       - Input: TaskWithDependencies[], maxConcurrency = 3
       - Output: ExecutionWave[]
       - Algorithm: Kahn's algorithm (topological sort using in-degree counting)
       - Steps:
         a. Build task map, in-degree map, and dependents (reverse) map
         b. While tasks remain: find all with in-degree=0, add to wave
         c. If no tasks have in-degree=0 but tasks remain -> circular dependency
         d. Slice wave tasks to maxConcurrency
         e. Update in-degrees for completed tasks' dependents
       - Return waves array

    6. WaveScheduler class:
       - Constructor takes SubAgentExecutor instance
       - executeWaves method:
         a. Takes tasks, context, options
         b. Calls computeExecutionWaves to get execution plan
         c. For each wave (sequential):
            - Execute tasks in parallel using Promise.all
            - Wait for all tasks in wave to complete
            - If any task fails, mark all dependent tasks in future waves as cancelled
            - Update previousResults Map after wave completes
         d. Return array of ScheduledTask with results

    Constants:
    - MAX_CONCURRENCY = 3 (fixed per CONTEXT.md decision)

    Create tests/lib/agents/scheduler.test.ts with comprehensive test coverage.
