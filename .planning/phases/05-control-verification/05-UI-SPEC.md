## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` / `oklch(1 0 0)` | Page background, main surfaces |
| Secondary (30%) | `bg-muted` / `oklch(0.97 0 0)` | Cards, task rows, Wave containers |
| Accent (10%) | `bg-primary` / `oklch(0.205 0 0)` | Primary action buttons, running status indicator |
| Destructive | `bg-destructive` / `oklch(0.577 0.245 27.325)` | Cancel button, failed status indicator |
| Warning | `bg-amber-600` / `oklch(0.68 0.17 58)` | Paused state indicator (per existing approval-prompt pattern) |

Accent reserved for:
- Pause button (when workflow is running)
- Resume button (when workflow is paused)
- Running task status indicator (blue-500 via TaskStatusIcon)
- Primary call-to-action buttons

Destructive reserved for:
- Cancel button (terminates entire workflow)
- Failed task status indicator (red-500 via TaskStatusIcon)

Warning reserved for:
- Paused workflow state badge
- Paused status indicator (amber color family, matching approval-prompt pattern)

## Interaction Patterns

### Workflow Control Buttons

**Location:** Above the Pipeline view, right-aligned in a button group.

**Layout:**
```
+------------------------------------------+
|  Wave 1  Wave 2  Wave 3    [Pause] [Cancel]
+------------------------------------------+
```

**Button Variants:**
| Button | Variant | Size | Icon |
|--------|---------|------|------|
| Pause | `outline` | `sm` | `Pause` from lucide-react |
| Resume | `default` | `sm` | `Play` from lucide-react |
| Cancel | `destructive` | `sm` | `X` from lucide-react |

**Visibility Rules:**
- Show **Pause** when `workflow.status === 'running'`
- Show **Resume** when `workflow.status === 'paused'`
- Show **Cancel** when `workflow.status === 'running' || workflow.status === 'paused'`
- Hide all buttons when `workflow.status === 'completed' || workflow.status === 'failed' || workflow.status === 'cancelled'`

### Workflow Status Badge

**Location:** Left of the control buttons, inline with the Wave header.

**States:**
| Status | Badge Color | Text |
|--------|-------------|------|
| running | `bg-blue-100 text-blue-700 dark:bg-blue-900/30 dark:text-blue-400` | "Running" |
| paused | `bg-amber-100 text-amber-700 dark:bg-amber-900/30 dark:text-amber-400` | "Paused" |
| completed | `bg-green-100 text-green-700 dark:bg-green-900/30 dark:text-green-400` | "Completed" |
| failed | `bg-red-100 text-red-700 dark:bg-red-900/30 dark:text-red-400` | "Failed" |
| cancelled | `bg-gray-100 text-gray-700 dark:bg-gray-900/30 dark:text-gray-400` | "Cancelled" |

### Result Display with Source Attribution

**Location:** Below each completed task row in the Pipeline view.

**Format:**
```tsx
<div className="text-xs text-muted-foreground">
  [{agentType.charAt(0).toUpperCase() + agentType.slice(1)} Agent] {skillId}
</div>
```

**Example Output:**
- `[File Agent] file-processing`
- `[Search Agent] web-search`
- `[Code Agent] code-generation`
- `[Custom Agent] my-custom-skill`

### Cancel Confirmation Dialog

**Trigger:** Click on Cancel button.

**Dialog Content:**
- Title: "Cancel Workflow"
- Body: "Are you sure? This will terminate all running and pending tasks. This action cannot be undone."
- Actions:
  - "No, keep running" (outline variant) - closes dialog
  - "Yes, cancel workflow" (destructive variant) - executes cancel

**Implementation:** Use existing `Dialog` component from `@/components/ui/dialog`.
