  <behavior>
    - Test 1: createApprovalRequest creates pending approval with unique ID
    - Test 2: approveRequest(id) changes status to approved and logs audit
    - Test 3: rejectRequest(id) changes status to rejected and logs audit
    - Test 4: getApprovalStatus(id) returns current status
    - Test 5: Pending approvals expire after timeout (default 5 minutes)
    - Test 6: Action budget is tracked per session
  </behavior>
  <action>
    Create src/lib/approval/state.ts with:

    1. ApprovalStatus type: 'pending' | 'approved' | 'rejected' | 'expired'

    2. ApprovalRequest interface:
       - id: string
       - skillId: string
       - skillName: string
       - action: string (human-readable description)
       - details: string (what will happen)
       - input: unknown (the actual input to pass if approved)
       - context: SkillContext
       - status: ApprovalStatus
       - createdAt: Date
       - expiresAt: Date

    3. ActionBudget interface:
       - sessionId: string
       - actionsExecuted: number
       - maxActions: number
       - lastResetAt: Date

    4. Constants:
       - DEFAULT_APPROVAL_TIMEOUT_MS = 5 * 60 * 1000 (5 minutes)
       - DEFAULT_MAX_ACTIONS_PER_SESSION = 100

    5. Global state (in-memory for MVP):
       - pendingApprovals: Map<string, ApprovalRequest>
       - sessionBudgets: Map<string, ActionBudget>

    6. Functions:
       - createApprovalRequest(skill: SkillMetadata, input: unknown, context: SkillContext): ApprovalRequest
         - Generate unique ID
         - Create request with pending status
         - Set expiry time
         - Store in pendingApprovals
         - Log audit event
         - Return request

       - approveRequest(id: string): Promise<{ success: boolean; request?: ApprovalRequest }>
         - Get request from pendingApprovals
         - If not found or not pending, return { success: false }
         - Update status to approved
         - Log audit with action: 'tool_approval'
         - Return { success: true, request }

       - rejectRequest(id: string): Promise<{ success: boolean; request?: ApprovalRequest }>
         - Similar to approve but status = rejected
         - Log audit with action: 'tool_rejection'

       - getApprovalStatus(id: string): ApprovalStatus | undefined

       - cleanupExpiredApprovals(): void
         - Remove expired approvals from pendingApprovals

       - checkActionBudget(sessionId: string): { allowed: boolean; remaining: number }
         - Get or create budget for session
         - Return whether action is allowed and remaining count

       - incrementActionCount(sessionId: string): void
         - Increment action count for session

    Export all types and functions.
  </action>
  <verify>
    <automated>npm test -- --run tests/lib/skills/approval.test.ts</automated>
  </verify>
  <done>
    - src/lib/approval/state.ts exports approval state machine
    - Tests pass: create, approve, reject, expire, budget tracking
