## Component Inventory

### FileChip (`src/components/files/file-chip.tsx`)

Compact inline chip showing file preview during upload.

**States:**

| State | Visual |
|-------|--------|
| `pending` | Muted background, type icon (FileText/FileCode/FileSpreadsheet), filename truncated, file size, close (X) button |
| `uploading` | Same as pending but with thin progress bar at bottom of chip (2px height), percentage text replacing file size, type icon replaced by spinner (Loader2 with animate-spin) |
| `uploaded` | Emerald-tinted type icon, filename, file size, close button |
| `error` | Destructive border (1px), AlertCircle icon, error reason text replacing file size, retry button (RefreshCw icon), auto-fade after 5 seconds via opacity transition |

**Layout:**
```
[Type Icon] [Filename (truncated, max-w-[160px])] [Size/Progress] [X/Retry]
```
- Container: `inline-flex items-center gap-1.5 px-2 py-1 rounded-md bg-muted text-xs`
- Progress bar: absolute bottom-0, left-0, height 0.5 (2px), rounded-full, width transitions via `transition-all duration-300`
- Filename: `truncate max-w-[160px]`
- Icon size: `h-3.5 w-3.5`
- Close button: ghost variant, icon-xs size, `ml-0.5`

**Auto-fade (error state):**
- After 5 seconds: `opacity-100` transitions to `opacity-0` over 500ms
- On fade complete: chip removed from DOM
- User can click retry or close before auto-fade

### FileUploadButton (`src/components/files/file-upload-button.tsx`)

Paperclip icon button positioned to the left of the Textarea.

**Visual:**
- shadcn Button, variant `ghost`, size `icon-sm` (28px)

- Color: `text-muted-foreground` default, `text-foreground` on hover
- Hidden `<input type="file" multiple>` triggered on click

**Hover/active animation (Claude's discretion -- default):**
- Hover: `hover:bg-muted` (from ghost variant)

- Focus: `focus-visible:ring-2 focus-visible:ring-ring` (from button base)

### useFileUpload Hook (`src/hooks/use-file-upload.ts`)

**Interface:**
```typescript
interface FileUploadState {
  files: PendingFile[];
  isUploading: boolean;
  addFiles: (fileList: FileList) => void;
  removeFile: (id: string) => void;
  retryFile: (id: string) => void;
  clearFiles: () => void;
  getUploadedFileIds: () => string[]; // returns IDs of successfully uploaded files
}
```

**Validation logic (client-side):**
1. Extension whitelist check (fast rejection)
2. File size check (100MB max)
3. File count check (5 max)
4. Returns error string or null

**Upload implementation:**
- Uses `XMLHttpRequest` (not fetch) for upload progress events
- Progress updates at ~5% granularity (every 5 percentage points)
- On success: stores server response metadata in `uploadedFile` field
- On error: sets `error` field with user-facing message

## Interaction Patterns

### Drag-and-Drop

| Event | Behavior |
|-------|----------|
| `dragenter` on form | Set `isDragging = true`, show overlay |
| `dragover` on form | `e.preventDefault()`, `e.stopPropagation()` (prevent browser default file open) |
| `dragleave` on form | Set `isDragging = false`, hide overlay (only if leaving the form, not entering a child) |
| `drop` on form | `e.preventDefault()`, extract files from `e.dataTransfer.files`, call `addFiles()`, set `isDragging = false` |

**Child element dragleave fix:** Use a counter approach or check `relatedTarget` to prevent flicker when dragging over child elements within the drop zone.

### File Validation Flow

```
User selects/drops file
  -> Client extension check
    -> FAIL: show error chip (no upload attempted)
    -> PASS: Client size check
      -> FAIL: show error chip ("File exceeds 100MB")
      -> PASS: File count check
        -> FAIL: show error chip ("Maximum 5 files per message")
        -> PASS: Start upload
          -> Server validates magic bytes
            -> FAIL: chip transitions to error state with server error message
            -> PASS: chip transitions to uploaded state
```

### Upload Progress

- Progress bar width: `${progress}%` of chip width
- Progress text: `${progress}%` displayed where file size normally goes
- Smooth transition: `transition-all duration-300 ease-out` on progress bar width
- At 100%: brief delay (200ms), then transition to uploaded state

### Error Auto-Fade

1. Error chip appears with full opacity
2. After 5 seconds, begin fade-out transition (500ms duration)
3. After fade completes (5.5s total), remove chip from array
4. User interactions (retry, close) cancel the auto-fade timer

## Accessibility

| Element | Requirement |
|---------|-------------|
| Paperclip button | `aria-label="Attach file"`, keyboard accessible (Enter/Space opens file picker) |
| File chips | `role="status"` with `aria-label="{filename} - {status}"` for screen reader announcements |
| Progress bar | `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `aria-label="Uploading {filename}"` |
| Error chips | `role="alert"` for immediate screen reader announcement |
| Retry button | `aria-label="Retry upload of {filename}"` |
| Close button | `aria-label="Remove {filename}"` |
| Drag overlay | `role="region"` with `aria-label="File drop zone"` |
| Hidden file input | `aria-hidden="true"`, `tabIndex={-1}` |
