## Preview Panel Specification

### Panel Header (D-07 from CONTEXT.md)

| Element | Value |
|---------|-------|
| Layout | Vertical stack: filename (large) + metadata row (type, size, time) + status bar |
| Filename | `text-base font-medium`, truncated with title attribute |
| Metadata row | `text-xs text-muted-foreground`, inline: type icon + type label + " | " + size + " | " + upload time |
| Status bar | Conditional rendering based on status |
| Retry button | When status is `failed`: Button `variant="outline" size="sm"` with RefreshCw icon + "重新提取" text, calls POST /api/files/[id]/extract |
| Delete button | Button `variant="ghost" size="icon-sm"` with Trash2 icon, triggers delete dialog |

### Preview Content Area

| File Type | Rendering | Source Field | Fallback |
|-----------|-----------|-------------|----------|
| document | react-markdown + remark-gfm | `extractedMarkdown` | "暂无内容" if null/empty |
| code | react-syntax-highlighter (Prism + oneLight) | `extractedContent` | "暂无内容" if null/empty |
| data | HTML table from JSON.parse(`extractedContent`) | `extractedContent` | "此文件无可预览的数据" if null/empty or parse fails |

### Document Preview Details

- Reuse exact same ReactMarkdown configuration from `chat-message.tsx` (remark-gfm plugin, code block syntax highlighting with Prism + oneLight)
- Wrap in `prose prose-zinc max-w-none` div (same as chat-message.tsx)
- Content area: `ScrollArea` with `overflow-y-auto`, fills remaining panel height

### Code Preview Details

- Language detected from filename extension using same `LANGUAGE_MAP` from `code-extractor.ts` (client-side inference)
- If language cannot be determined, default to `plaintext`
- `react-syntax-highlighter` with `oneLight` style, `PreTag="div"`
- Show filename as a tab/header above the code block: `bg-muted px-3 py-1.5 text-xs font-mono text-muted-foreground rounded-t-lg`
- Wrap in `ScrollArea` with `overflow-x-auto overflow-y-auto`

### Data Preview Details

- Parse `extractedContent` as JSON (it is `JSON.stringify(rows)` where rows is `Record<string, unknown>[]`)
- If parse fails, show error message: "文件内容格式异常，无法渲染表格。"
- Render as HTML table with zebra striping (`even:bg-muted/30`)
- First row keys become column headers
- Max 100 rows displayed with "显示前 100 行，共 {total} 行" footer if total > 100
- Wrap in `ScrollArea` with `overflow-x-auto overflow-y-auto`

### Failed Extraction Preview

When status is `failed`:
- Header shows errorMessage from the file record
- Retry button is visible in header
- Content area shows: AlertCircle icon + "内容提取失败：{errorMessage}"

Source: CONTEXT.md D-06 (preview rendering strategy), D-07 (preview panel header). ReactMarkdown/SyntaxHighlighter patterns from RESEARCH.md code examples and existing chat-message.tsx. Data table and code tab styling are Claude's discretion.

## Interaction Patterns

### File Selection Flow

| Trigger | Behavior |
|---------|----------|
| User clicks a table row | Row becomes selected (accent background + left border), right panel fetches file detail via `use-file-detail` hook, preview renders |
| User clicks a different row | Previous selection clears, new row selected, preview updates |
| User clicks same row | No change (stays selected) |

### Delete Flow

| Trigger | Behavior |
|---------|----------|
| User clicks Trash2 icon on a row | FileDeleteDialog opens with file context |
| User clicks "取消" | Dialog closes, no action |
| User clicks "删除" | Button shows spinner, DELETE /api/files/[id] called |
| Delete succeeds | Dialog closes, file removed from table list, preview panel clears (shows initial state if deleted file was selected) |
| Delete fails | Error toast "删除失败，请重试。", dialog stays open |

### Retry Extraction Flow

| Trigger | Behavior |
|---------|----------|
| User clicks "重新提取" in preview header | POST /api/files/[id]/extract called, button shows Loader2 spinner |
| Retry succeeds | File status updates to `processing` then `ready` via polling, preview re-renders with content |
| Retry fails | Status stays `failed`, errorMessage updates, preview shows new error message |

### Filter Change Flow

| Trigger | Behavior |
|---------|----------|
| User clicks a filter button | Active button style changes, file list refetches with new filter, pagination resets to page 1, preview panel clears if current file is filtered out |

### Pagination Flow

| Trigger | Behavior |
|---------|----------|
| User clicks Prev/Next | File list refetches with new page, preview panel clears |
| User on last page deletes last file | Auto-navigate to previous page |

Source: CONTEXT.md D-04 (sort/pagination), D-05 (filter), D-08 (delete). Interaction flows are Claude's discretion.

| Element | Requirement |
|---------|-------------|

## API Integration Points

### GET /api/files

**Purpose:** Fetch paginated, sorted, filtered file list.

**Query parameters:** `page`, `pageSize`, `sortBy`, `sortOrder`, `fileType`

**Expected response:**
```json
{
  "files": [
    {
      "id": "uuid",
      "filename": "report.pdf",
      "mimeType": "application/pdf",
      "size": 1024000,
      "fileType": "document",
      "status": "ready",
      "createdAt": "2026-03-27T10:00:00Z",
      "errorMessage": null
    }
  ],
  "total": 42,
  "page": 1,
  "totalPages": 3
}
```

**Called by:** `useFileList` hook

**Note:** Does NOT return `extractedContent` or `extractedMarkdown` (those are large fields fetched separately for preview). Source: RESEARCH.md Pitfall 1.

### GET /api/files/[id]

**Purpose:** Fetch full file detail including extracted content for preview.

**Expected response:**
```json
{
  "id": "uuid",
  "filename": "report.pdf",
  "mimeType": "application/pdf",
  "size": 1024000,
  "fileType": "document",
  "status": "ready",
  "createdAt": "2026-03-27T10:00:00Z",
  "errorMessage": null,
  "extractedContent": "...",
  "extractedMarkdown": "# Report\n..."
}
```

**Called by:** `useFileDetail` hook (fetched when a file is selected for preview)

### DELETE /api/files/[id]

**Purpose:** Delete a file (both DB record and storage file).

**Expected response:**
```json
{ "success": true }
```

**Called by:** FileDeleteDialog on confirm
