## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` / `oklch(1 0 0)` | Page background, main surfaces |
| Secondary (30%) | `bg-muted` / `oklch(0.97 0 0)` | Cards, agent rows, containers |
| Accent (10%) | `bg-primary` / `oklch(0.205 0 0)` | Progress bar fill, running indicator, expand toggle |
| Destructive | `bg-destructive` / `oklch(0.577 0.245 27.325)` | Failed status, error messages |
| Success | `bg-green-600` / `oklch(0.627 0.194 149)` | Completed status, success indicators |
| Warning | `bg-amber-600` / `oklch(0.68 0.17 58)` | Paused state indicator |

Accent reserved for:
- Progress bar fill (workflow completion)
- Running task status indicator (blue-500 via TaskStatusIcon)
- Expand/collapse toggle icons
- Active agent highlight

Destructive reserved for:
- Failed task status indicator
- Error messages in log viewer

Success reserved for:
- Completed task status indicator
- 100% progress bar fill state

## Interaction Patterns

### Real-time Agent Status Display (VIS-01, VIS-02)

**Location:** Embedded in chat message area, below the user's task request.

**Layout:**
```
+----------------------------------------------------------+
| [Status Badge: Running]         Progress: 2/4 (50%)      |
+----------------------------------------------------------+
| Wave 1                                                    |
|   [completed icon] Process PDF files                      |
|       [File Agent] pdf-extractor                          |
|   [completed icon] Search documentation                   |
|       [Search Agent] web-search                           |
+----------------------------------------------------------+
| Wave 2 (Running)                                          |
|   [running icon] Generate code                            |
|       [Code Agent] code-generation     [Show Logs]        |
+----------------------------------------------------------+
```

**Components:**
- AgentStatusList: Container for all waves and tasks
- AgentTaskRow: Individual task with status, description, source label
- CollapsibleLogSection: Expandable log viewer per task

### Workflow Progress Indicator (VIS-03)

**Location:** Above the agent list, inline with status badge.

**Visual:**
```
Progress: [=========>----------] 2/4 (50%)
```

**Progress Bar Specification:**
- Height: 8px (h-2 in Tailwind)
- Background: `bg-muted` (secondary)
- Fill: `bg-primary` (accent)
- Border radius: `rounded-full`
- Animation: `transition-all duration-300` on width change

**States:**
| Completion | Bar Color | Text |
|------------|-----------|------|
| 0-99% | `bg-primary` | "{completed}/{total} ({percentage}%)" |
| 100% | `bg-green-600` | "All tasks completed (100%)" |
| Failed | `bg-destructive` | "Workflow failed" |

### Execution Log Viewer (VIS-04)

**Trigger:** Click "Show Logs" button on a task row.

**Behavior:**
- Expands inline below the task row
- Shows timestamped execution logs
- Auto-scrolls to latest entry when expanded
- Maximum height: 200px with `overflow-y-auto`

**Log Entry Format:**
```
[14:32:15] Agent started execution
[14:32:18] Calling skill: pdf-extractor
[14:32:45] Skill completed successfully
[14:32:46] Returning results to orchestrator
```

**Typography:**
- Font: `font-mono` (monospace)
- Size: `text-xs` (12px)
- Color: `text-muted-foreground`

**Log Entry Types:**
| Type | Color | Icon |
|------|-------|------|
| info | `text-muted-foreground` | none |
| success | `text-green-600` | `CheckCircle` |
| error | `text-destructive` | `XCircle` |
| warning | `text-amber-600` | `AlertTriangle` |

### Compact Display Pattern (VIS-05)

**Constraint:** Process information must not overwhelm the main conversation.

**Implementation:**
- Default: Collapsed view showing only status badge and progress
- Expandable: User clicks to see detailed agent list
- Max width: Same as chat message width
- Visual hierarchy: Status > Progress > Agent list (hidden by default)

**Collapsed State:**
```
[Running] 2/4 tasks (50%)                    [Expand]
```

**Expanded State:**
```
[Running] 2/4 tasks (50%)                    [Collapse]
+--------------------------------------------------------+
| (Full agent list and progress details)                 |
+--------------------------------------------------------+
```

## SSE Real-time Updates

**Endpoint:** `/api/workflow-status?workflowId={id}` (existing)

**Message Format:**
```json
{
  "type": "task_update",
  "taskId": "task-123",
  "status": "running",
  "progress": 50,
  "log": {
    "timestamp": "2026-03-26T14:32:15Z",
    "level": "info",
    "message": "Agent started execution"
  }
}
```

**Update Handling:**
1. Update task status in AgentStatusList
2. Update progress bar percentage
3. Append log entry if panel is expanded
4. Play subtle animation on status change
