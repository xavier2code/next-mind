**1. Create `src/app/api/health/route.ts`:**
- Import `NextResponse` from `'next/server'`
- Import `db` from `'@/lib/db'`
- Import `{ sql }` from `'drizzle-orm'`
- Export `async function GET()`:
  - try/catch block:
    - try: `await db.execute(sql\`SELECT 1\`);` then return `NextResponse.json({ status: 'ok', db: 'connected', timestamp: new Date().toISOString() })` (status 200, default)
    - catch: return `NextResponse.json({ status: 'error', db: 'disconnected', timestamp: new Date().toISOString() }, { status: 503 })`
- Keep it simple -- no additional dependencies, no logging, no auth checks

**2. Update `src/middleware.ts`:**
- Add a new check for `/api/health` EARLY in the middleware function, BEFORE the `isProtectedApiRoute` check:
  - Add `const isHealthRoute = req.nextUrl.pathname === '/api/health';` after the existing variable declarations (after line 17)
  - Modify the first condition block (currently `if (isAuthPage || isApiAuthRoute)`) to become `if (isHealthRoute || isAuthPage || isApiAuthRoute)` -- this routes /api/health through the same "allow" path as auth routes
  - The existing logic inside this block already calls `NextResponse.next()` for non-auth-page routes, so health will pass through without auth
  - IMPORTANT: Do NOT change any other logic in the middleware. The auth page redirect, API protection, and chat page protection must remain unchanged.

**3. Update `docker-compose.yml`:**
- Add a `healthcheck` block to the `app` service, AFTER the `volumes` section:
  ```yaml
  healthcheck:
    test: ["CMD-SHELL", "wget -qO- http://localhost:3000/api/health || exit 1"]
    interval: 10s
    timeout: 5s
    retries: 5
    start_period: 30s
  ```
  Use `wget` (not `curl`) because the Docker image is based on Node-alpine which includes BusyBox wget by default. The `-qO-` flags: `-q` quiet, `-O-` output to stdout. Returns exit 0 on success, non-zero on failure.
  </action>
